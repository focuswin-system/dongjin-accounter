## 1. 개요

### 1.1 목적

현재 프로토타입은 거래내역·계약·급여·결의서 등 기본 골격은 갖췄으나, 실무 회계 워크플로우에서 필수적인 기능이 누락되어 있다.  
이번 v2에서는 회계 담당자 인터뷰를 바탕으로 확인된 **6개 기능 영역**을 추가하여, 실제 업무 사용 가능한 수준으로 끌어올린다.

### 1.2 배경

동진테크는 방산 납품 중심 제조업으로, 계약 기반 수익 구조를 갖는다.  
주요 문제점:

- **현금 위치를 모른다**: 어느 계좌에 돈이 얼마 있는지 시스템으로 확인 불가
- **미수금 체계가 틀렸다**: 청구서 발행 없이 거래내역 상태만으로 미수금을 판단하는 구조
- **부가세 신고 자료가 없다**: 매출·매입 세금계산서 집계 화면이 없어 분기마다 수작업
- **반복 지출을 매번 수동 입력**: 임차료·보험료 등 정기 지출 자동화 없음
- **계약-수금이 단절**: 계약 마일스톤이 있어도 수금 예정이 자동 생성되지 않음
- **원가 관리가 없다**: 수주 시 예상 원가가 없어 계약별 수익성 관리 불가

### 1.3 관련 문서

- 현재 코드베이스: `src/screens/`, `src/lib/data.js`, `src/lib/api.js`
- 기존 데이터 구조: `SAMPLE.incomes`, `SAMPLE.receivables`, `SAMPLE.expenses`

## 2. 범위

## 3. 기능 요구사항

### 3.1 F1 — 계좌별 잔액 관리

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F1-01 | 등록된 계좌별 현재 잔액 표시 (초기잔액 + 입금 합계 - 출금 합계) | High |
| F1-02 | 수동 조정 기능: 조정 금액, 조정 사유, 조정 일자 입력 | High |
| F1-03 | 조정 이력 목록 (감사 추적) | High |
| F1-04 | 계좌별 월별 입출금 합계 요약 | Medium |
| F1-05 | 홈 대시보드 자금 현황 카드에 계좌별 잔액 반영 | Medium |

**핵심 로직**:
```
계좌 잔액 = 초기잔액(수동 설정)
           + Σ 해당 계좌 입금 거래
           - Σ 해당 계좌 출금 거래
           + Σ 해당 계좌 조정 금액
```

---

### 3.2 F2 — 청구 관리 메뉴

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F2-01 | 청구서 발행 등록 (발주처, 계약, 청구 금액, 청구일, 지급 기한) | High |
| F2-02 | 청구서 수취 등록 (협력사, 계약/범위, 금액, 수취일, 지급 기한) | High |
| F2-03 | 청구서와 실제 입출금 거래 매칭 기능 | High |
| F2-04 | 미매칭 청구서 → 미수금(발행)/미지급금(수취) 자동 산출 | High |
| F2-05 | 일부 입금 처리: 청구액 - 입금액 = 잔여 미수금 | High |
| F2-06 | 청구 목록 필터: 기간, 거래처, 계약, 상태 | Medium |
| F2-07 | 기존 거래내역 탭의 미수금/미지급금을 청구 메뉴 기반으로 전환 | High |

**상태 흐름**:
```
청구서 발행
  → [입금 미확인] → 거래 매칭 → [입금 완료]
                              → [일부 입금] (잔여 미수금 유지)
  → [기한 지남] (due date 경과 시 자동)
  → [장기 미수] (30일+ 연체)
```

**미수금 산출 로직**:
```
미수금 = Σ (청구금액 - 매칭된 입금금액)  // 미매칭 또는 일부 매칭 청구서
미지급금 = Σ (수취청구금액 - 매칭된 출금금액)
```

---

### 3.3 F3 — 부가세 관리

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F3-01 | 매출 세금계산서 목록 (발행일, 공급가액, 세액, 거래처, 연결 청구서) | High |
| F3-02 | 매입 세금계산서 목록 (수취일, 공급가액, 세액, 거래처, 연결 결의서) | High |
| F3-03 | 분기별 매출세액 / 매입세액 / 납부(환급) 예상액 집계 | High |
| F3-04 | 기간 선택 (1분기~4분기, 직접 입력) | Medium |
| F3-05 | 세금계산서 미등록 거래 경보 (증빙 누락과 연동) | Medium |

**집계 로직**:
```
매출세액 = Σ 발행 세금계산서 세액
매입세액 = Σ 수취 세금계산서 세액
납부세액 = 매출세액 - 매입세액  (음수면 환급)
```

### 3.4 F4 — 정기 지출 스케줄

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F4-01 | 정기 지출 등록 (거래처, 금액, 반복 주기, 시작일, 종료일) | High |
| F4-02 | 반복 주기: 매월/매분기/매년 | High |
| F4-03 | 매월 자동으로 "지급 예정" 상태 거래내역 생성 | High |
| F4-04 | 금액 변경 예외 처리 (특정 월만 다른 금액) | Medium |
| F4-05 | 정기 지출 목록 및 활성/비활성 관리 | Medium |

**대상 정기 지출 예시**:
- 공장 임차료 (매월 3,200,000원)
- 4대보험 (매월 변동, 급여 연동)
- 통신비, 전기료 등

### 3.5 F5 — 수금 예정 자동 생성

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F5-01 | 계약 등록 시 마일스톤 설정 (선급금/기성고/잔금 비율 + 예정일) | High |
| F5-02 | 마일스톤 기반 수금 예정 자동 생성 (청구 메뉴에 "청구 예정"으로) | High |
| F5-03 | 납품 완료 이벤트 → 해당 마일스톤 수금 예정 활성화 | Medium |
| F5-04 | 홈 대시보드 "입금 예정" 목록 자동 반영 | High |
| F5-05 | 수금 예정일 D-7, D-3, D-0 알림 | Medium |

**마일스톤 구조 예시**:
```
계약 KF-21 동체 부품 (142,000,000원)
  ├── 선급금  20% = 28,400,000  2026-01-30  [완료]
  ├── 기성고1 30% = 42,600,000  2026-03-15  [완료]
  ├── 기성고2 30% = 42,600,000  2026-04-20  [완료]
  └── 잔금   20% = 28,400,000  2026-05-20  [예정] ← 자동 생성
```

### 3.6 F6 — 계약별 원가 예산

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F6-01 | 계약 등록/편집 시 원가 예산 입력 (항목별: 재료비/외주비/인건비/경비) | High |
| F6-02 | 실제 지출 거래 연동 → 항목별 실적 자동 집계 | High |
| F6-03 | 예산 대비 실적 현황 (금액 + 달성률 %) | High |
| F6-04 | 원가율 = 총 원가 / 계약금액 (목표 원가율 vs 실제) | High |
| F6-05 | 원가 초과 경보 (항목별 80% 도달 시 경고, 100% 초과 시 알림) | Medium |
| F6-06 | 계약 화면 수익성 섹션에 예산/실적/원가율 표시 | High |

**원가율 계산**:
```
목표 원가율 = 원가 예산 합계 / 계약금액
실제 원가율 = 실제 지출 합계 / 계약금액
예상 이익   = 계약금액 - 실제 지출 합계
```

## 5. 성공 기준

### 5.1 완료 정의

- [ ] 6개 기능 영역 모두 화면 구현 완료
- [ ] 홈 대시보드 숫자가 각 기능 데이터와 연동됨
- [ ] 미수금/미지급금이 청구 메뉴 기반으로 자동 산출됨
- [ ] 계좌별 잔액이 거래내역 기반으로 집계됨
- [ ] 계약 마일스톤 → 수금 예정 자동 생성 동작

### 5.2 데모 시나리오 (담당자 검토용)

1. 계좌 잔액 확인 → 수동 조정 → 조정 이력 확인
2. 청구서 발행 등록 → 입금 거래 매칭 → 미수금 변화 확인
3. 부가세 메뉴에서 이번 분기 납부 예상세액 확인
4. 정기 지출 등록 → 다음 달 "지급 예정" 자동 생성 확인
5. 계약에 마일스톤 입력 → 홈 "입금 예정" 자동 반영 확인
6. 계약 원가 예산 입력 → 지출 입력 → 원가율 변화 확인

## 7. 아키텍처 고려사항

### 7.1 프로젝트 레벨

| 레벨 | 선택 |
|------|:----:|
| Starter | ☐ |
| **Dynamic** | ☑ |
| Enterprise | ☐ |

현재 구조(React + Vite + 단일 파일 컴포넌트)를 유지하되, 신규 기능은 각각 독립 스크린으로 분리.

### 7.2 화면 구조 변경

```
현재 네비게이션
├── 홈
├── 거래내역 (미수금/미지급금 탭 포함)
├── 계약
├── 인사관리
├── 보고서
└── 설정

v2 네비게이션
├── 홈
├── 거래내역
├── 청구 관리          ← 신규 (F2)
│   ├── 발행 청구서 (미수금)
│   └── 수취 청구서 (미지급금)
├── 계약
│   └── [계약 상세] 원가 예산 섹션 추가 (F6)
├── 인사관리
├── 보고서
│   └── 부가세 탭 추가 (F3)
└── 설정
    └── [계좌 관리] 잔액 + 조정 추가 (F1)
        [정기 지출] 신규 탭 추가 (F4)
        [수금 예정] 마일스톤 설정 (F5, 계약과 연동)
```

### 7.3 데이터 구조 변경 (`data.js`)

```js
// 신규 추가
ACCOUNTS_BALANCE: [
  { id: "acc-1", name: "기업은행 *123", initialBalance: 50000000,
    adjustments: [{ date, amount, reason, by }] }
]

INVOICES: [  // F2 청구서
  { id, kind: "issued"|"received", vendor, contract,
    amount, vat, issuedAt, dueAt,
    status: "pending"|"partial"|"matched"|"overdue",
    matches: [{ txnId, amount }] }
]

RECURRING_EXPENSES: [  // F4
  { id, vendor, category, amount, period: "monthly"|"quarterly",
    startDate, endDate, active: true }
]

// 기존 변경
CONTRACT_LIST: [
  { ...기존,
    costBudget: { material, outsource, labor, overhead },  // F6 신규
    milestones: [{ type, ratio, amount, dueDate, status }] // F5 신규
  }
]
```
