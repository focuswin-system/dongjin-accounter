## 발견된 문제

### [HIGH] #1 — 처리 시 날짜 입력 불가 (입금·지급 공통)
- **위치**: 홈 > 오늘 할 일 > [처리] 버튼 / [이체] 버튼
- **문제**: 입금 확인·이체 실행 모달에 날짜 입력 없음. 어제 들어온 입금을 오늘 처리하면 날짜가 오늘로 고정됨. 지급(이체)도 동일.
- **영향**: 원장 날짜 오류 → 월별 집계 부정확
- **수정안**: 모달에 날짜 입력 필드 추가 (기본값: 오늘). 입금/이체 모두 적용.

### [HIGH] #2 — 지출 등록 진입점 혼란 (세금계산서 흐름 우회 가능)
- **위치**: 거래내역 > 거래 등록 > 지출 등록
- **문제**: 세금계산서 있는 지출(외주가공비·재료비 등)도 원장에 직접 등록 가능 → 미지급금 추적 안 됨, 부가세 매칭 안 됨
- **영향**: 미지급금 숫자 부정확, 부가세 신고 자료 누락
- **수정안**: 지출 성격을 첫 단계에서 분리
  - 세금계산서 있는 지출 → 청구 관리(수취 청구서) 등록 유도
  - 소액/영수증 지출 → 거래내역 직접 등록 (현재 방식 유지)

### [MED] #3 — 지출 드로어 "계약/공통" 필드 맥락 힌트 부족
- **위치**: 거래내역 > 지출 등록 드로어 > 계약/공통 필드
- **문제**: "계약/공통"이 뭘 뜻하는지 처음 사용자는 모름. 어떤 계약에 귀속되는지 vs 공통비인지 구분이 불명확.
- **수정안**: 플레이스홀더 또는 하위 설명 추가 ("이 지출이 귀속될 납품 계약 선택. 특정 계약 귀속 아니면 공통 선택")

### [MED] #4 — 결제수단 '계좌이체'일 때 '사용 직원' 필드 불필요
- **위치**: 거래내역 > 지출 등록 드로어 > 결제수단 → 계좌이체 선택 시
- **문제**: 계좌이체는 법인 계좌에서 나가는 돈인데 '사용 직원' 필드가 계속 보임. 사용 직원은 개인 법인카드·현금 지출에만 의미 있음.
- **수정안**: 결제수단이 '계좌이체'이면 '사용 직원' 필드 숨김. 법인카드·현금 선택 시에만 표시.

### [MED] #5 — 공급가액 직접 입력 모드 부재
- **위치**: 거래내역 > 지출 등록 드로어 > 금액 입력
- **문제**: 세금계산서 기반 지출은 공급가액(VAT 제외)이 기준. 현재 총액(VAT 포함) 입력만 지원. 공급가액 10,000,000원짜리 세금계산서를 처리하려면 11,000,000원 입력 후 수동 계산 필요.
- **수정안**: "공급가액 입력" 토글 추가 → 공급가액 입력 시 VAT 10% 자동 계산 및 합계 표시

### [MED] #7 — 청구서 입금 처리 시 원장 연동 + 입금일 입력 없음
- **위치**: 청구 관리 > 청구서 상세 > 입금 처리 버튼
- **문제**: 청구서에서 입금 처리해도 원장(거래내역)에 자동 수금 내역이 생성되지 않음. 날짜 입력도 없음.
- **수정안**: 입금 처리 모달에 입금일 필드 추가 + 원장에 수금 거래 자동 생성 (계약·청구서 연결)

### [MED] #8 — 청구서 수정·삭제 진입점 없음
- **위치**: 청구 관리 > 청구서 카드
- **문제**: 청구서를 잘못 등록해도 수정·삭제 버튼이 안 보임. 현재 카드 UI에 액션 없음.
- **수정안**: 청구서 카드 우측에 더보기(…) 메뉴 추가 → 수정 / 삭제 / 입금 처리 액션

### [LOW] #6 — 지출 등록 후 목록 즉시 미반영 (mock 상태 관리)
- **위치**: 거래내역 > 지출 등록 드로어 저장 후
- **문제**: 저장 후 드로어가 닫히지만 목록에 새 항목이 바로 나타나지 않을 수 있음 (mock 상태 관리 이슈)
- **수정안**: 백엔드 연동 시 자동 해결 예정. 현재는 무시.

### [LOW] #9 — 정기 지출 "다음 생성" 날짜 off-by-one 오류
- **위치**: 설정 > 정기 지출 목록 > 다음 생성 컬럼
- **문제**: 다음 생성 날짜가 실제 기준일보다 1일 앞당겨 표시됨
  - KT 통신비: 매월 25일인데 2026-05-24로 표시
  - 한전 전력비: 매월 15일인데 2026-06-14로 표시
  - 4대보험: 매월 10일인데 2026-06-09로 표시
  - 임차료: 매월 1일인데 2026-05-31로 표시 (6월 1일이어야 함)
- **수정안**: "다음 생성" 날짜 계산 로직에서 -1일 offset 제거
